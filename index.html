Mental Models

- Every concept in our head has some sort of mental models attached to it - Our brain forms mental models implicitly - But it's often helpful to make our model models more explicit - This can help us discover where our models of the world break down and where they don't - We just saw a Python oddity that tested our mental models of Python's variables and data structures

- We could try to mentally model Python's variables and data structures as buckets that contain objects - If we assign a variable to an empty list **(click)**, that variable contains that list **(click)** - Then when we assign another variable to the first one, **(click)** the list is copied into a new bucket **(click)** - Let's say we then append `9` to that first list **(click)**, which would **(click)** mutate the first list - But the second list shouldn't change... **(click)** - ... Except it *does* **(click)** change! - This mental model **(click)** isn't going to work - Variables in Python are *not* like buckets that contain objects - Let's try another one...

>>>
- Let's mentally model variables as sticky notes on objects - When we assign a variable to an empty list **(click)**, we attach that variable name to the object **(click)** - Then when we assign **(click)** another variable to the first one, we simply **(click)** attach *another name* to the same object - So when we change **(click)** that first list, the first one changes **(click)** but the second one changes also **(click)**... because it's all just one list - This mental model works pretty well! - But what if...

- Let's mentally model variables and data structures as references or pointers - When we **(click)** assign a variable to an empty list, we *point* **(click)** that variable to the object - Then when we assign **(click)** another variable to the first one, we point **(click)** the second variable to the same object - If we **(click)** modify the first list **(click)**, the first list changes **(click)** and so does the second **(click)** because *they're the same list* - And if we **(click)** put our list inside another list, we'll simply be **(click)** pointing one of the indexes in that new list to *our* existing list - This mental model *is* a bit more complex than the sticky note model, but it stretches a bit further - The indirection between variables and objects is tricky, but it also makes it easier to understand how data structures work - I'd argue that this mental model is the closest one to how Python actually models variables and objects - But keep in mind that *every* mental model has its limits

Confirmation Bias

Validating your current assumptions

- Confirmation bias - Which involves seeking out information that confirms your existing assumptions or beliefs - This happens when we try to validate our assumptions instead of trying to invalidate them

Anchoring Bias

- There's also the anchoring bias - which occurs when our brains inadvertently anchor our thinking on a specific piece of information, which can sometimes cloud our judgement - For example, when Steve Jobs, announced the price of the iPad, **(click)** he displayed $999 on the screen for many seconds before eventually announcing that the price of an iPad would be $499 **(click)** - That made the iPads *immediately* seem inexpensive - This doesn't just apply to prices though...
